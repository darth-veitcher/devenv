Comprehensive documentation for both the devenv cookiecutter template AND the projects it generates.

- **No MkDocs infrastructure** - No mkdocs.yml in either root or template

  - mkdocstrings[python]>=0.25.0

## Technical Decisions

### Theme: mkdocs-material
- Modern, feature-rich theme
- Dark/light mode toggle
- Search, code copy, navigation features
- Wide adoption in Python community

### API Documentation: mkdocstrings
- Auto-generates from docstrings
- Google-style docstring format
- Shows source code (configurable)
- Links between references

### Hosting: GitHub Pages Ready
- `mkdocs gh-deploy` command included
- Can also use ReadTheDocs
- Works with GitHub Actions CI

### Docstring Style: Google
```python
def example(param: str) -> bool:
    """Short description.

    Longer description if needed.

    Args:
        param: Description of parameter.

    Returns:
        Description of return value.

    Raises:
        ValueError: When param is invalid.
    """
```

## Dependencies

- Bullet 3 depends on 1-2 (need mkdocs working before mkdocstrings)
- Bullet 6 depends on 3 (need mkdocstrings for reference/)

## Order of Execution

Template docs FIRST (Bullets 1-6) because:
1. More complex (mkdocstrings integration)
2. Users benefit immediately when generating projects
3. Root docs can reference "what you'll get"
